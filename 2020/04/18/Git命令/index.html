Git命令（持续更新）
Created 2020-04-18| Updated 2020-05-02
Post View:

这里记录下的都是些关于git的基本操作命令：
低阶命令：
 clean：清除屏幕内容；
 ll： 查看当前目录下的文件详细信息；
 find：查看当前目录文件；
 find 目录名 -type f：查看指定目录下的文件(具体到目录下的子文件夹下的文件)；
 echo “xxx” > 文件名：将内容”xxx”写入文件当中，相当于创建了一个文件；
 vim 文件名(带有扩展名):用vim编辑器打开
  按i进入编辑；按esc退出编辑；
  在非编辑模式下，按:set nu可以显示行数；
  退出编辑后按:wq即为保存文件并退出vim编辑器；
  退出编辑后还可以按:q!强制退出但不保存文件内容；
 git hash-object -w 文件路径名：将文件内容以键值对的git对象保存到git版本库当中；
 git cat-file -p 哈希名：根据键值查看被压缩的详细内容；
 git cat-file -t 哈希名：根据哈希名查看存储的对象类型；
 git update-index ––add ––cacheinfo 100644/100755/120000 \哈希名 文件名：为文件的首个版本创建一个暂存区
  ––add：文件不在暂存区时，首次加入需要下上––add，若文件内容已被修改而暂存区中已有文件则不需要加––add，内容被修改后的文件会覆盖原来在暂存区的文件；
  ––cacheinfo：添加的文件在git数据库中，而不在当前目录，所以需要添加––cacheinfo；
  100644：普通文件；100755：可执行文件；120000：符号链接
 git ls-files -s：查看暂存区的文件；
 git write-tree：生成树对象；
 git read-tree ––prefix=bak 哈希：读取第一个树对象；
 echo “注释” | git commit-tree 哈希：提交对象，就是将用户对树对象的注解注释与树对象一并封装提交到git版本库当中；

高阶命令：
 git init：初始化仓库；
 git status：查看文件状态；
 git add 路径：将工作为项目提交到暂存区（执行流程：在版本库中生成git对象->将git对象也放入暂存区中）；
 git commit -m “注释内容”：创建提交对象并放入版本库中（执行流程：暂存区中的git对象变为树对象放入版本库->版本库中对应的树对象与注释内容一把封装为一个提交对象->将提交对象放入版本库）
 git deff：查看已修改且没有放入暂存的文件修改内容；
 git diff ––cache/git diff ––staged：查看已修改且放入暂存区的文件的修改内容；
 git commit：在执行git add命令后，为树对象添加注释内容，这是跳转到一个类似于vim编辑器的界面，让用户添加内容（回车后树对象与注释封装为提交对象提交到版本库）；
 git commit -a -m “注释内容”：跳过存入暂存区（git add命令）放入git版本库（使用此命令的前提条件是文件必须是被add过的）；
 mv 文件原名 文件新名：修改文件名（仅仅是从工作目录中删除文件，这个操作并没有提交到git数据库中，需要另外执行add和commit命令）；
 rm 文件名：删除文件（仅仅是从工作目录中删除文件，这个操作并没有提交到git数据库中，需要另外执行add和commit命令）；
 git log：查看文件历史记录；
 git log ––pretty=online：查看文件历史记录（漂亮的）；
 git log ––online：查看文件历史记录（简略的）；
 git rm 文件名：从工作区中删除文件且将记录提交到git数据库中；
 git mv 文件原名 文件新名：修改文件名且将记录提交到git数据库中；
 分支部分：
 git branch 分支名：创建分支；
 git branch：查看所以分支；
 git branch -D 分支名：强制删除分支；
 git checkout 分支名：切换分支；
 git branch -v：查看分支最后的提交对象；
 git branch 分支名 提交对象的哈希：创建新分支并使得这个新分支指向某个提交对象；
 git log ––oneline ––decorate ––graph ––all：查看项目分支历史；
 git config ––global alias.命令别名 “命令”：为长或难记的命令设置别名，命令部分的书写要求：原命令去掉git剩余部分，剩余部分只是一个单词的可以不用加引号，否则都要加；
 git checkout -b 分支名：新建一个分支并切换到新分支；
 git merge 分支名：合并分支（必须切换到master上进行合并）；
 git stash：将未完成的修改保存在一个栈上，进而可以不同提交修改而切换到另外一个分支；
 git stash apply 栈名（例如：stash@{number}）：重新应用已保存在栈里的修改；
 git stash drop 栈名（例如：stash@{number}）：删除指定的栈；
 git stash pop：应用栈的储藏并删掉栈；
 git checkout ––filename：在工作区中撤销文件的修改；
 git reset HEAD filename：撤回保存到暂存区的修改；
 git commit ––amend：修改提交到版本库的文件的注释并覆盖修改之前的注释；
