MySQL数据类型---set（集合）与enum（枚举）
Created 2020-04-13| Updated 2020-04-23
Post View:

set（集合）：
（1）创建表时，set类型的数据，值要初始化，不然没法使用；
（2）在表中插入一条数据时，set类型的字段的数据要包含至少一个你所初始的set里的数据，例如：
 先建立一个非常非常简单的表，kind字段里的数据已经set为3个；
 
 接着是插入数据，运行成功；
 
 在看下表里的数据，显示刚才插入的数据，点击省略号也可以查看到之前set好的选项内容；

 
 尝试下插入一个不是在set集合里的数据，显示运行成功，但是；
 
 看回表里的数据，在kind那列确实空白，因为插入了一个非set集合里的数据；
 
 试着插入多个数据；
 
 成了！表的数据更新，插入成功！
 

enum（枚举类型）：
（1）跟set一样，枚举类型的数据，值也是要初始化；
（2）往表里插入数据时，数据类型为枚举的字段，插入的数据需要是枚举类型中的其中一种，插入非枚举类型或多种值会报错，例如：
 接着上面创建的表加入数据类型为enum的字段sex，并设置好它的值；
 
 插入数据；
 
 数据表数据更新，成功显示（因为之前的几行数据没有更新，所以sex字段显示null，没有影响，忽视就ok）；
 
 看看插入一个sex字段数据为“男”，看看什么效果；
 
 数据表的数据没有更新，sex字段一列显示空白！插入的数据不在枚举范围内，显示不成功
 
 你所插入的数据需要是男/女。

  set（集合）和enum（枚举）两者看起来很像，但实际使用会有很不一样的效果。
